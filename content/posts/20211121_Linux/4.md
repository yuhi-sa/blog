# トラブルシューティング
一般的に下位のレイヤから確認を行う．
1. 物理的な確認
2. IPアドレスの設定確認
3. ネットワーク経路確認
4. 開いているポートの確認

# 代表的なコマンド
- ping：宛先へのネットワークの疎通確認
- telnet：アプリケーション層での疎通確認
- traceroute：始点から終点までのホスト間の経路情報確認
- ip route：ルーティングテーブルのエントリ情報の表示
- ss：ネットワーク接続の統計情報の出力(socket statistics)
- tcpdump：ネットワーク上を流れる通信パケットの閲覧
